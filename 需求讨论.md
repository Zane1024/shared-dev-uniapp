# 共享设备管理平台需求文档

## 1. 项目概述
本项目是一个共享设备管理平台，适用于共享充电宝、共享雨伞、共享设备等场景。主要载体为微信小程序，支持多角色用户体系，提供设备管理、场地管理、经营统计、积分商城等核心功能。

## 2. 用户角色
- **普通会员**：使用设备、兑换积分商品、查看个人消费记录
- **管理员**：管理设备、场地、查看经营数据、配置系统参数

## 3. 核心功能模块

### 3.1 登录认证模块
- **账号密码登录**：传统登录方式
- **微信授权登录**：一键授权，登录后需绑定手机号并设置密码
- **登录后流程**：
  1. 首次微信登录需获取用户手机号
  2. 绑定手机号码
  3. 设置登录密码（用于后续账号密码登录）

### 3.2 首页仪表板
- **头部销售看板**：
  - 今日销售情况（醒目显示）
  - 时间筛选器（支持按日、周、月筛选）
- **收益统计卡片**：
  - 今日投币金额
  - 广告收益
  - 累计投币总额
  - 累计总收益
- **核心功能入口**：
  - 经验统计（数据分析）
  - 场地管理
  - 绑定设备（进入设备绑定流程）
- **公告区域**：
  - 跑马灯公告展示系统通知

### 3.3 设备管理模块
- **设备列表**：展示所有设备状态（在线/离线/使用中/故障）
- **设备详情**：设备基本信息、使用记录、维护记录
- **设备绑定流程**：
  1. 进入绑定界面显示使用守则
  2. 扫一扫功能扫描设备二维码
  3. 确认绑定设备信息
- **设备状态监控**：实时监控设备运行状态

### 3.4 场地管理模块
- **场地列表**：展示所有投放场地信息
- **场地详情**：场地位置、设备数量、负责人信息
- **场地统计**：各场地收益对比、设备使用率

### 3.5 经营统计模块
- **数据概览**：关键指标展示（用户数、订单数、收益等）
- **趋势分析**：收益趋势、用户增长趋势

### 3.6 积分商城模块
- **商品列表**：展示所有可兑换商品
- **商品详情**：商品信息、所需积分、库存状态
- **兑换流程**：
  1. 选择商品直接兑换（无购物车）
  2. 填写收货地址
  3. 积分支付扣减
  4. 生成订单
- **订单管理**：
  - 订单列表（待发货/已发货/已完成）
  - 订单详情（订单状态、物流信息）

## 4. 页面流程设计

### 4.1 管理员端和用户端流程
```
登录/注册 → 首页仪表板 → [设备使用/积分商城/] → 个人中心 → [设备管理/场地管理/经营统计]
```

## 5. 技术实现建议

### 5.1 技术栈选择（基于现有tdesign-uniapp-starter项目）
- **前端框架**：uni-app + Vue 3.x（实际使用Vue 3.4.21）
- **UI组件库**：@tdesign/uniapp（版本0.7.3，优先使用）
- **开发语言**：TypeScript（已配置TypeScript支持）
- **HTTP客户端**：基于现有`src/api/request.ts`封装扩展
- **状态管理**：建议使用Pinia（更适合Vue 3），当前项目可集成
- **构建工具**：Vite 5.2.8 + @dcloudio/vite-plugin-uni
- **包管理器**：pnpm 10.27.0（首选）
- **代码规范**：ESLint + Stylelint + Husky + lint-staged（已预配置）
- **CSS预处理器**：Less（已配置，使用`src/styles/variable.less`管理变量）

### 5.3 关键组件与TDesign使用建议

#### 5.3.1 基础布局组件
- **页面布局**：使用`t-container`、`t-header`、`t-main`、`t-footer`构建页面框架
- **导航栏**：使用`t-navbar`定制顶部导航，结合`t-tab-bar`实现底部导航
- **卡片容器**：使用`t-card`包装内容区域，统一卡片样式

#### 5.3.2 数据展示组件
- **统计卡片**：结合`t-card` + `t-row`/`t-col`栅格系统展示数据指标
- **数据图表**：使用`t-chart`或集成第三方图表库（如ECharts）展示经营数据
- **列表展示**：使用`t-list`展示设备/场地/商品列表，支持下拉刷新和上拉加载

#### 5.3.3 表单与交互组件
- **登录表单**：使用`t-form` + `t-input`构建登录界面，支持验证规则
- **筛选器**：使用`t-picker`时间选择器、`t-select`下拉选择实现数据筛选
- **扫码功能**：使用uni-app的`uni.scanCode` API，配合`t-button`触发

#### 5.3.4 业务特定组件
1. **设备状态卡片**：
   - 使用`t-card` + `t-tag`展示设备状态（在线/离线/使用中）
   - `t-button`提供操作按钮（绑定/解绑/详情）
   - `t-progress`展示设备使用率

2. **商品展示卡片**：
   - `t-card`展示商品图片和基本信息
   - `t-tag`标记商品库存状态
   - `t-button`实现"立即兑换"操作

3. **数据看板组件**：
   - `t-row`/`t-col`栅格系统排布统计卡片
   - `t-statistic`展示数值数据（今日收益、累计投币等）
   - `t-date-picker`提供时间筛选功能

4. **跑马灯公告**：
   - 使用`t-notice-bar`实现公告滚动效果
   - 支持点击跳转和自动轮播

#### 5.3.5 反馈与提示组件
- **加载状态**：使用`t-loading`展示数据加载状态
- **消息提示**：使用`t-toast`、`t-dialog`、`t-message`提供用户反馈
- **空状态**：使用`t-empty`展示无数据时的友好提示

### 5.4 接口设计要点
- 用户认证接口（登录、注册、绑定手机号）
- 设备管理接口（设备列表、状态更新、绑定解绑）
- 场地管理接口（场地CRUD、场地统计）
- 订单管理接口（订单创建、查询、状态更新）
- 数据统计接口（经营数据查询、报表生成）

## 7. 微信小程序集成要点

### 7.1 微信API使用
- **登录授权**：使用`uni.login()`获取code，后端换取openid和session_key
- **获取手机号**：使用`<button open-type="getPhoneNumber">`获取加密数据，后端解密
- **扫码功能**：使用`uni.scanCode()`扫描设备二维码
- **支付功能**：使用`uni.requestPayment()`调用微信支付
- **分享功能**：配置`onShareAppMessage`实现页面分享

### 7.2 小程序配置
- **appid配置**：在`manifest.json`中配置微信小程序appid
- **权限配置**：在`manifest.json`中声明所需权限（用户信息、地理位置等）
- **分包加载**：对于大型项目，考虑使用分包加载优化首屏性能
- **版本管理**：使用小程序CI工具（已配置`mp:upload`脚本）进行版本上传

### 7.3 平台差异处理
- **条件编译**：使用`#ifdef MP-WEIXIN`处理小程序特有逻辑
- **API兼容**：注意uni-app API在小程序端的支持情况
- **样式适配**：小程序部分CSS特性受限，需测试验证

## 8. 注意事项
1. **微信小程序限制**：注意微信API调用限制和审核要求，避免使用受限API
2. **数据安全**：用户手机号等敏感信息需后端加密存储，传输使用HTTPS
3. **性能优化**：列表数据分页加载，图片懒加载，合理使用setData减少数据传输
4. **多端适配**：主要针对微信小程序开发，但保留H5端的兼容性考虑
